## 2. Repo Structure

```
morning/
├── briefs/
│   └── YYYY-MM-DD.html     ← one file per day
├── index.html              ← always the latest brief (copy of most recent)
├── archive.html            ← browseable index of all past briefs
├── vercel.json             ← static deployment config
├── CONTEXT.md              ← this file
└── README.md
```

**After generating a brief:**
1. Save to `briefs/YYYY-MM-DD.html`
2. `cp briefs/YYYY-MM-DD.html index.html`
3. Add entry to `archive.html` (top of current month section)
4. `git add . && git commit -m "Brief YYYY-MM-DD · [one-line headline]" && git push`

**To push with auth:**
```bash
git remote set-url origin https://TOKEN@github.com/marcioherlein/morning.git
git push origin main
```

## 6. Style & Tone Rules

- **Direct, no sugarcoating.** Call bad news clearly. Marcio explicitly wants raw honesty.
- **Actionable first.** Every section has a portfolio "so what."
- **Cite inline** as `<a href="URL">source</a>` — never footnotes.
- **Always numbers.** % changes, bps, dollar levels. No "rose significantly."
- **Bold the lead** in every paragraph for scannability.
- **War day counter** in header: "War Day 26"
- **Probabilities** in scenario boxes — update from news, always ~100% total.
- **Footer format:** `Generated by Claude · [Weekday] [Date] · [Time] ART · War Day [N]`
